# GitGuide 产品介绍

## 一句话介绍

**GitGuide｜边点边学 GitHub 是一个面向 GitHub 新手的中英文双语交互式学习网站，通过模拟 GitHub 使用场景和任务式引导，帮助用户边点边学会使用 GitHub。**

英文版：

> **GitGuide is a bilingual interactive learning platform that helps beginners learn GitHub through guided simulations and task-based practice.**

---

# 1. 产品名称

## GitGuide

标准对外表达：

* **GitGuide｜边点边学 GitHub**

标准 Slogan：

> **GitGuide｜边点边学 GitHub**

---

# 2. 产品定位

GitGuide 不是 GitHub 的复制品，也不是传统图文教程。

它的定位是：

> **面向零基础用户的 GitHub 双语交互式导航训练平台。**

GitGuide 通过接近 GitHub 使用场景的模拟页面，把复杂的 GitHub 操作拆成一个个简单任务，让用户在安全、可重复练习的环境中学会 GitHub。

---

# 3. 产品背景

很多编程初学者第一次打开 GitHub 时，会遇到这些问题：

* 不知道 GitHub 是做什么的。
* 看不懂 Repository、Fork、Clone、Issue、Pull Request 等英文术语。
* 不知道 README、Code、Issues、Pull Requests 等页面入口有什么作用。
* 不知道如何搜索项目。
* 不知道如何下载代码。
* 不知道如何创建自己的仓库。
* 害怕在真实 GitHub 上点错、创建错、操作错。
* 看文章和视频教程时觉得懂了，但真正打开 GitHub 后还是不会操作。

GitGuide 要解决的就是这个问题：

> **让新手先在模拟环境里点一遍、练一遍，再去真实 GitHub 使用。**

---

# 4. 产品目标

GitGuide 的核心目标是帮助用户完成从“看不懂 GitHub”到“能独立完成基础操作”的转变。

具体目标包括：

1. 帮助用户理解 GitHub 的基本页面结构。
2. 帮助用户掌握常见英文术语。
3. 引导用户完成浏览仓库、搜索项目、下载代码、创建仓库等基础任务。
4. 降低中文用户学习 GitHub 的语言门槛。
5. 提供安全的模拟练习环境，减少真实操作压力。
6. 让用户通过任务式学习获得即时反馈。
7. 最终引导用户进入真实 GitHub 场景。

---

# 5. 目标用户

## 5.1 编程初学者

这类用户刚开始学习编程，听说过 GitHub，但不知道怎么使用。

他们需要：

* 看懂 GitHub 页面。
* 学会搜索项目。
* 学会下载代码。
* 学会创建自己的仓库。

---

## 5.2 学生和培训班学员

这类用户可能需要用 GitHub 交作业、看课程代码或提交项目。

他们需要：

* 快速掌握基础操作。
* 按步骤完成任务。
* 理解老师提到的 GitHub 术语。
* 有中文引导降低学习难度。

---

## 5.3 开源新手

这类用户想参与开源项目，但不知道从哪里开始。

他们需要：

* 学会浏览开源项目。
* 学会阅读 README。
* 学会查看 Issue。
* 学会理解 Fork 和 Pull Request 的基本流程。

---

## 5.4 非英语母语用户

这类用户可能有一定技术基础，但对 GitHub 英文界面不够熟悉。

他们需要：

* 中英文术语对照。
* 英文按钮解释。
* GitHub 页面元素说明。
* 用中文理解，用英文适应真实界面。

---

# 6. 核心价值

GitGuide 的核心价值可以概括为：

> **把 GitHub 的复杂操作变成可视化、可练习、可完成的小任务。**

具体价值包括：

## 6.1 降低门槛

用户不需要一开始就注册 GitHub、安装 Git、配置 SSH，也可以先理解 GitHub 的基本操作。

---

## 6.2 交互式学习

用户不是只看教程，而是在模拟页面里真正点击、搜索、选择、填写和完成任务。

---

## 6.3 双语引导

GitGuide 提供中文和英文版本，帮助用户理解 GitHub 的英文界面，同时建立技术英语词汇基础。

---

## 6.4 安全练习

用户在模拟环境中操作，不会影响真实 GitHub 账号，也不会误删或误改真实项目。

---

## 6.5 任务驱动

每个学习内容都被拆成明确任务，例如：

* 找到 README。
* 点击 Code 按钮。
* 下载 ZIP。
* 复制 Clone 地址。
* 创建一个仓库。
* 查看 Issue。

用户完成任务后获得反馈，更容易建立信心。

---

# 7. 核心功能概览

## 7.1 中英文切换

GitGuide 支持中文和英文两种语言。

用户可以在学习过程中随时切换语言，查看术语解释和操作说明。

---

## 7.2 学习路径

GitGuide 提供结构化学习路径，例如：

1. 认识 GitHub
2. 浏览仓库
3. 搜索项目
4. 下载代码
5. 创建仓库
6. 了解 Issue
7. 了解 Fork 和 Pull Request

---

## 7.3 模拟 GitHub 页面

GitGuide 会提供接近 GitHub 使用场景的模拟页面，例如：

* 模拟仓库首页
* 模拟搜索结果页
* 模拟代码下载菜单
* 模拟创建仓库页面
* 模拟 Issue 页面

这些页面用于教学，不是 GitHub 官方页面，也不替代 GitHub。

---

## 7.4 任务式引导

系统会通过高亮、提示气泡、步骤说明和完成反馈引导用户操作。

示例：

> 第 1 步：点击绿色的 Code 按钮
> 第 2 步：查看 Download ZIP 选项
> 第 3 步：点击 Download ZIP
> 第 4 步：完成代码下载任务

---

## 7.5 术语解释

用户可以查看 GitHub 常见术语的中英文解释，例如：

| 英文术语 | 中文解释 | 含义 |
| --- | --- | --- |
| Repository | 仓库 | 存放代码、文档和项目历史的地方 |
| README | 项目说明 | 介绍项目用途、安装和使用方法的文档 |
| Issue | 问题 / 议题 | 用于反馈问题、讨论需求或记录任务 |
| Fork | 派生 | 把别人的仓库复制到自己的账号下 |
| Clone | 克隆 | 把远程仓库复制到本地电脑 |
| Pull Request | 拉取请求 | 请求把代码修改合并到项目中 |

---

## 7.6 学习进度记录

GitGuide 可以记录用户已完成的任务，让用户知道自己学到了哪里。

例如：

* 已完成 3 / 8 个任务
* 当前正在学习“下载代码”
* 下一个任务是“创建仓库”

---

## 7.7 从模拟到实战

当用户完成模拟任务后，GitGuide 可以引导用户进入真实 GitHub 完成同样操作。

例如：

> 你已经学会了如何下载代码。现在可以前往 GitHub，搜索一个真实项目并尝试下载。

---

# 9. 产品差异化

GitGuide 与传统教程的区别：

| 传统教程 | GitGuide |
| --- | --- |
| 阅读文章 | 通过操作学习 |
| 看视频 | 亲自点击练习 |
| 术语解释分散 | 中英文术语集中解释 |
| 直接进入真实 GitHub | 先在模拟环境练习 |
| 用户容易迷路 | 每一步都有引导 |
| 学完不一定会操作 | 每个任务都有完成反馈 |

GitGuide 的优势在于：

> **它不是告诉用户“GitHub 怎么用”，而是带用户真的用一遍。**

# 10. 使用场景

## 场景 1：零基础用户第一次认识 GitHub

用户进入 GitGuide，选择“认识 GitHub”，通过模拟仓库页面了解仓库、README、文件列表和 Code 按钮。

---

## 场景 2：学生需要下载课程代码

用户选择“下载代码”任务，系统引导用户点击 Code 按钮，理解 Download ZIP 和 Clone URL 的区别，最终学会下载代码。

---

## 场景 3：用户想创建自己的第一个项目

用户选择“创建仓库”任务，在模拟表单中填写仓库名称、描述、可见性，并完成创建流程。

---

## 场景 4：中文用户适应英文 GitHub 界面

用户在中文说明下学习 Repository、Issue、Fork、Pull Request 等英文术语，逐步适应真实 GitHub 的英文界面。

# 12. 最终产品介绍文案

## 中文正式版

**GitGuide｜边点边学 GitHub 是一个面向 GitHub 新手的中英文双语交互式学习网站。**

它通过模拟 GitHub 的常见使用场景，将浏览仓库、搜索项目、下载代码、创建仓库等操作拆解成一个个简单任务，并通过高亮提示、步骤引导和即时反馈，帮助用户边点边学、边练边懂。

GitGuide 的目标不是替代 GitHub，而是帮助用户在进入真实 GitHub 之前，先理解页面结构、熟悉常见术语、掌握基础操作。对于编程初学者、学生、开源新手和非英语母语用户来说，GitGuide 提供了一个安全、友好、低压力的 GitHub 学习环境。

---

## 英文正式版

**GitGuide is a bilingual interactive learning website for GitHub beginners.**

It uses guided simulations to break common GitHub workflows, such as browsing repositories, searching projects, downloading code, and creating repositories, into simple step-by-step tasks. With highlighted elements, contextual hints, and instant feedback, GitGuide helps users learn GitHub by doing.

GitGuide is not a replacement for GitHub. Instead, it helps users understand the interface, learn common terminology, and practice essential workflows before moving into real GitHub scenarios. It is designed for programming beginners, students, open-source newcomers, and non-native English speakers who want a safer and more beginner-friendly way to learn GitHub.
